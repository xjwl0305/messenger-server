## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Installation

```bash
$ yarn install
```

## Running the app

```bash
# development
$ yarn run start

# watch mode
$ yarn run start:dev

# production mode
$ yarn run start:prod
```

## Test

```bash
# unit tests
$ yarn run test

# e2e tests
$ yarn run test:e2e

# test coverage
$ yarn run test:cov
```

## Description

임의의 네트워크 프로토콜 (HTTP, TCP, UDP 등등) 을 통해 클라이언트로부터 메시지를 받아 적재한다.
메신저 서버는 여러 개의 단톡방을 지원해야 한다. 하나의 메시지는 하나의 단톡방에 들어간다.
서버는 클라이언트로부터 메시지를 받은 이후 최소 24시간동안 저장해야 한다. 
클라이언트가 특정 단톡방에 대한 message를 요청하면, 해당 클라이언트가 마지막으로 읽은 시점 이후에 단톡방에 들어온 메세지를 들어온 순서대로 제공해야 한다. 단, 시간이 지나 서버에서 사라진 메세지는 보여줄 필요가 없다.
처음 단톡방에 들어온 클라이언트에게는 서버에 저장되어 있는 단톡방 내의 모든 메세지를 제공한다.
여러 클라이언트가 동시에 한 단톡방 안에서 메세지를 보내거나 받을 수 있다. 
단톡방 내의 메세지는 실수로 사라지면 오해의 소지가 있기 때문에 최대한 메세지 유실이 없어야 한다.
